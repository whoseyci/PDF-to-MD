## Notes on pipeline (v2)

Built on `pymupdf4llm` + custom postprocessing. Key fixes vs v1:

- ✅ `<br>`-stacked tables reshaped into proper multi-row markdown tables
- ✅ Picture-text blocks (OCR'd from table-images) rendered as fenced code blocks
- ✅ Three citation styles linked: `(Author, Year)`, `Author (Year)`, `[N]`
- ✅ Numeric citation linking only fires when refs ARE numbered (no false positives on author footnotes)
- ✅ Multi-cite syntax: `(Smith, 2024; Jones, 2025)` → each linked separately
- ✅ Better title extraction (skips journal banners; first real heading)
- ✅ Authors detected as a list, footnote markers stripped
- ✅ Page header/footer deduplication (running titles removed)
- ✅ DOI auto-detected and rendered as clickable link
- ✅ Junk tables filtered (title blocks, author lists, single-cell pseudo-tables)
- ✅ Large PDFs (>100 pages) processed in chunked mode (memory-safe)
- ✅ Resumable: re-running skips already-done papers

## Known limitations

- Picture-text blocks (OCR'd tables) are preserved as fixed-width text;
  full structured reconstruction without original column positions is unreliable.
  The corresponding figure PNG is always kept too.
- Multi-language papers (translated abstracts in Spanish/Chinese) may show
  coverage > 1 or < 1 because pymupdf4llm extracts only Latin scripts cleanly.
- Citation linking quality depends on a clean references section; some
  papers use atypical formats and a portion of cites remain unlinked.
